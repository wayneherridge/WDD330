Transforms and Transitions - CSS3

This is another way pages can be styled without a line of Javascript

Transforms

This property translates, rotates, scales, and/or skews any element on the page. Elements can be easily manipulated.

Translation

This allows elements to be moved left, right, up, or down.

A translated element can only be moved relative to its curent position.

    
        transform: translate(45px, -45px);

This (translate(x,y)) moves an element x from the left, and y from the top.

translateX(45px) and translateY(45px) moves 45px to the right along the x axis and 45px up along the y axis.

Scaling

scaling(x,y) scales horizontally and vertically.

Rotation

rotate(10deg) rotates an element 10deg clockwise.

Skew

skew(x,y) allows elements to be skewed along the x and y axis.

Changing the Origin of the Transform

The origin of the transform can be controlled using the transform-origin property.

Transitions

Transitions allow the values of CSS properties to change over time, essentially providing simple animations.

Steps to creating a simple transition:

  1. Declare the original state of the element in the default style declaration.
  2. Declare the final state of your transitioned element; for example, a :hover state.
  3. Include the transition functions in your default style declaration using the transition properties, including: transition-property, transition-duration, transition-timing-function, and transition-delay. We’ll look at each of these and how they work shortly.

Animations

CSS animations, unlike transitions, allow you to control each step of an animation via keyframes.

This is much the same as FLASH works - with keyframes

Keyframes

To animate an element in CSS, you first create a named animation, then attach it to an element in that element’s property declaration block. Animations in themselves don’t do anything; in order to animate an element, you’ll need to associate the animation with that element.

Canvas, SVG, and Drag and Drop

Canvas

With HTML5’s Canvas API, we can draw anything we can imagine, all through JavaScript.

The canvas API is supported in these browsers:

  • Chrome 4+
  • Firefox 2+
  • Opera 9.6+
  • Safari 3.1+
  • iOS 3.2+
  • IE 9.0+
  • Android 3.0+

This can be used to create shapes, colors and text etc.

            
                function drawCircle(canvas) {
                var context = canvas.getContext("2d");
                context.beginPath();
                context.arc(100, 100, 50, 0, Math.PI*2, true);
                context.closePath();
                context.strokeStyle = "red";
                context.fillStyle = "blue";
                context.lineWidth = 3;
                context.fill(); 
                context.stroke(); 
                }

SVG

SVG stands for Scalable Vector Graphics, a specific file format that allows you to describe vector graphics using XML. A major selling point of vector graphics in general is that, unlike bitmap images (such as GIF, JPEG, PNG, and TIFF), vector images preserve their quality even as you blow them up or shrink them down.

SVG is a lot more complicated to write - especially for complicated shapes.

Drag and Drop

This API allows us to specify that certain elements are draggable, and then specify what should happen when these draggable elements are dragged over or dropped onto other elements on the page.

To make elements draggable we need to add the draggable attribute to them - please see Dev Tools for the JS code.
