Forms

Forms are made up of elements such as input fields, select menus and buttons.

The data entered into forms was traditionally sent and processed to a backend language like PHP or Ruby. It is possible now to process the information on the from end before sending it to the server using JS.

When building a form the designer/creator must have design principles at the forefront of their mind. Your form must give the user a good experience.

Form Events

Some of the events that forms are able to trigger are exclusive to forms.

The focus event occurs when an element is focused on.

            
                const input = form.elements.searchInput;
                input.addEventListener('focus', () => alert('focused'), false);

When the cursor is placed inside the the element (input) either by clicking or navigating to it on the keyboard it will show the message 'focused' on the screen.

When the cursor is placed in the input field the message should appear.

Submitting a Form

A submit event occurs when a form is submitted.

The content of the form is usually sent to the server to be processed but Javascript can be used to intercept the form before it is sent.

            
                const form = document.forms['search'];
                form.addEventListener ('submit', search, false);
                
                function search() {
                    alert('Form Submitted');
                }

When the submit button is clicked a message appears which says 'Form Submitted.

When we use the preventDefault() method it stops the form from being submitted to the URL.

            
                function search(event) {
                    alert('Form Submitted');
                    event.preventDefault();
                }

When an attempt is made to submit the form the message still appears but the form does not try and submit itself to the dummy URL.

Retrieving and Changing Values From a Form

The value property within an input element can be sued to retrieve text inside the field.

            
                function search(event) {
                    alert(`You Searched for: ${input.value}`);
                    event.preventDefault();
                }

Input is the variable that points to the input element in the form.

Other Types of Form Controls
Input Fields
  • Text Input Fields - used for short text (username etc) - type='text'
  • Password Input Fields - used to enter passwords or secret information - type='password'
  • Checkbox Input Fields - used to select different options at the same time - type='checkbox'
  • Radio Button Input Fields - only one of the options shown can be selected - type='radio'
  • Hidden Input Fields - not displayed by the browser but is given a value - submitted with the form - type='hidden'
  • File Input Types - used for uploading files - type='file'
  • Other Input Types - number, tel, color
  • Select Drop-Down List - a drop down list which can allow for more than one option to be selected - select and option
  • Text Areas - a textarea element is used to enter a block of text - textarea is the element name.
  • Buttons - Default type is submit. Reset is also an option which deletes the content of the inputs of the form. Button is the other type - not attached to anything and can be standalone.
Form Validation

This goes through the form and checks to make sure everything is correct.

The types of validation are:

  1. A require field - check that it has been completed
  2. An email address - check that what has been enetered is a valid format
  3. A number - check that the content is only numeric
  4. A password - check that the rules that have been set for a password has been followed

These items can be added using HTML but there is also a way of implementing validation using Javascript.

This way of validation is not standard so you would have to make up your own rules.

Object-Oriented Programming

The three main concepts in OOP are: encapsulation, polymorphism and inheritance.

Encapsulation

The inner workings are kept hidden inside the object and only the essential functionalities are exposed to the end user.

Polymorphism

The same process can be used on for different objects. In OOP various objects can share the same method but can also override shared methods with a more specific implementation.

Inheritance

This takes an object that already exists and inherits all of its properties and methods.

Static Methods

These can be used in class declarationns to create a static method or class methods. A static method is called by the class rather than by instances of the class.

            
                class Dice {
                    constructor(sides=6) {
                        this.sides = sides;
                    }
                    roll() {
                        return Math.floor(this.sides * Math.random() + 1)
                    }
                    static description() {
                        return 'A way of choosing random numbers'
                    }
                }

This method is called from the Dice class - Dice.description()

Static methods are not available to instances of the class. So, in our example, the instances of Dice such as redDice and blueDice cannot call the static description() method:

redDice.description

Getters & Setters

    
        me.age = 21;
        me.retirementAge = 65;
        Object.defineProperty(me, 'yearsToRetirement',{
            get() {
                if(this.age > this.retirementAge) { return 0; }
                else { return this.retirementAge - this.age; }
            },
            set(value) {
                this.age = this.retirementAge - value;
                return value;
            }
        });

if we add age and retirementAge properties to the me object, we can then create a yearsToRetirement property that depends on these properties

The get() and set() methods can be used to control how a property is set using assignment and the value that is returned when a property is queried.

The getter bases the yearsToRetirement property on the age and retirementAge properties
